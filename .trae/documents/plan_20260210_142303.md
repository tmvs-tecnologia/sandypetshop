# Plano de Atualização: Cards de Mensalistas

## 1. Ajuste de Condomínio/Serviço
- Modificar a lógica de exibição do condomínio:
  - Se o campo `condominium` estiver vazio, nulo ou for igual a 'Nenhum Condomínio', exibir "Banho & Tosa Fixo".
  - Caso contrário, exibir o nome do condomínio.

## 2. Exibição do Próximo Agendamento
- O código atual usa uma função placeholder `getNextAppointmentDateText` que retorna "Verificar Calendário".
- Vamos implementar uma lógica mais precisa para calcular a próxima data com base na recorrência (semanal, quinzenal ou mensal) e no dia da semana/mês configurado no cliente.
- Utilizaremos a data atual como referência para encontrar a próxima ocorrência válida.

## 3. Data do Próximo Pagamento (Fim do Mês)
- A regra de negócio solicitada é que a data de pagamento seja sempre o **último dia do mês atual**.
- Substituiremos a lógica atual (que usa dia 10 fixo ou data do cliente) por uma função que calcula dinamicamente o último dia do mês corrente (considerando anos bissextos, como 2028, etc., embora o exemplo do usuário mencione 2026).
- A exibição será formatada como `DD/MM/YYYY`.

## 4. Implementação
- Editar `src/components/MonthlyClientCard.tsx`.
- Atualizar a função `getNextAppointmentDateText` com lógica real de recorrência.
- Atualizar a função `getCurrentMonthPaymentDueISO` (ou criar uma nova `getLastDayOfCurrentMonth`) para retornar o último dia do mês.
- Ajustar o render do label de condomínio.

## 5. Verificação
- Verificar se clientes sem condomínio mostram "Banho & Tosa Fixo".
- Verificar se a data de pagamento mostra 28/02/2026 (ou data correspondente ao mês atual do sistema).
- Verificar se o próximo agendamento mostra uma data futura coerente com a regra de recorrência.
